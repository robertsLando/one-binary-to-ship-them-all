Enter Node.js SEA

Single Executable Applications

Official node:sea API — Node.js 20+

The promise: use stock Node.js binaries
to create standalone executables

But...

SEA's Limitations

  • ✗ Single file only — must pre-bundle everything into one .js file
  • ✗ No Virtual Filesystem — no fs.readFileSync for embedded files
  • ✗ No native addons — .node files need manual extraction boilerplate
  • ✗ No source protection — source code stored in plaintext, trivially extractable
  • ✗ No compression — assets stored uncompressed in the binary
  • ✗ No dependency walking — you figure out what to bundle

Filling the Gap PR #229

Enhanced SEA mode with walker integration & VFS

  • ✓ Dependency Walker — automatic discovery of all files & dependencies
  • ✓ Transparent VFS — fs, require, import work seamlessly on virtual files
  • ✓ Native Addon Support — auto-detect, embed & extract .node files at runtime
  • ✓ Worker Threads — VFS-aware workers with automatic bootstrap injection
  • ✓ Stock Node.js Binaries — no patched binaries needed. Uses official node:sea
  • ✓ ESM Native — works with ESM projects without CJS transform

The Key Insight

SEA + bundle: 39% faster build than PKG + bundle

PKG + bundle: smallest binary & fastest startup

SEA no bundle: native ESM — no transform needed

Bonus: useCodeCache — SEA + bundle + code cache: startup on par with PKG
Caveat: breaks cross-compilation (platform-specific) • Without bundle: only applies to entrypoint, limited benefit

The real win: stock Node.js binaries
Traditional mode still shines for bytecode protection & compression
